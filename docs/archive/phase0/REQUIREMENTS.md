# 要件定義書

## 1. 機能要件

### 1.1 ページ構成

| ID | 要件 | 優先度 |
|----|------|--------|
| F-001 | トップページに日記のタイムラインを表示する | 必須 |
| F-002 | 各日記エントリにタイトル・日付・本文を含める | 必須 |
| F-003 | 日記エントリを新しい順に表示する | 必須 |

### 1.2 テーマ・デザイン

| ID | 要件 | 優先度 |
|----|------|--------|
| F-010 | ダークテーマを既定のテーマとする | 必須 |
| F-011 | 背景色は暗色（黒〜濃灰）を使用する | 必須 |
| F-012 | テキストは明色で視認性を確保する | 必須 |

### 1.3 タイムライン表示

| ID | 要件 | 優先度 |
|----|------|--------|
| F-020 | 縦方向のタイムライン形式で日記を表示する | 必須 |
| F-021 | タイムライン上に日付マーカーを表示する | 推奨 |
| F-022 | エントリ間を線で接続し時系列を視覚化する | 推奨 |

### 1.4 アニメーション

| ID | 要件 | 優先度 |
|----|------|--------|
| F-030 | ページ読み込み時にフェードインアニメーションを適用する | 推奨 |
| F-031 | スクロール時にエントリが順次表示されるアニメーションを実装する | 推奨 |
| F-032 | ホバー時にインタラクティブな視覚フィードバックを提供する | 任意 |

### 1.5 レスポンシブデザイン

| ID | 要件 | 優先度 |
|----|------|--------|
| F-040 | モバイル端末（幅 〜767px）で適切に表示する | 必須 |
| F-041 | タブレット端末（幅 768px〜1023px）で適切に表示する | 必須 |
| F-042 | デスクトップ（幅 1024px〜）で適切に表示する | 必須 |

## 2. 非機能要件

### 2.1 技術制約

| ID | 要件 |
|----|------|
| NF-001 | HTML5 と CSS3 のみで実装する（JavaScriptフレームワーク不使用） |
| NF-002 | 外部CSSフレームワークを使用しない（バニラCSS） |
| NF-003 | 開発サーバーは Python `http.server` を使用する |

### 2.2 パフォーマンス

| ID | 要件 |
|----|------|
| NF-010 | 初回表示を3秒以内に完了する |
| NF-011 | 外部依存ライブラリを最小限に抑える |

### 2.3 互換性

| ID | 要件 |
|----|------|
| NF-020 | Chrome・Firefox・Safari の最新版で動作する |
| NF-021 | iOS Safari・Android Chrome で閲覧可能とする |

### 2.4 アクセシビリティ

| ID | 要件 |
|----|------|
| NF-030 | セマンティックHTMLを使用する |
| NF-031 | 十分なカラーコントラスト比を確保する（WCAG AA 基準） |

## 3. 画面設計方針

### 3.1 カラーパレット（ダークテーマ）

| 用途 | 色 |
|------|-----|
| 背景（メイン） | `#0d1117` 〜 `#161b22` |
| 背景（カード） | `#21262d` |
| テキスト（メイン） | `#e6edf3` |
| テキスト（補助） | `#8b949e` |
| アクセント | 任意（プロジェクトに合わせて選定） |

### 3.2 タイポグラフィ

- 本文: システムフォントスタック（sans-serif）
- 見出し: 本文と同系統、ウェイトで差別化
