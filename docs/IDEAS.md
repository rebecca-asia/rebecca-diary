### [2026-02-13] Apple Silicon温度取得（IOKit直接呼び出し）

**概要:**
`osx-cpu-temp`（brew）や `powermetrics`（sudo必須）に頼らず、Python の `ctypes` 経由で IOKit を直接呼び出してApple Silicon の温度データを取得する。

**動機:**
- Phase 1 では温度がオプショナル（osx-cpu-temp 未インストール時は null）
- sudo 不要で温度が取れればヘルスダッシュボードの完成度が上がる
- 外部依存ゼロの原則を維持できる

**課題:**
- IOKit の Apple Silicon 向け API は undocumented
- ctypes でのブリッジが複雑
- macOS バージョン間の互換性

**発見経緯:** Phase 1 レビューで powermetrics が sudo 必須であることを確認した際に浮上

### [2026-02-13] プロセス数・ロードアベレージをヘルス指標に追加

**概要:**
`top` の出力に含まれる Load Average（1/5/15分平均）と総プロセス数を、追加のヘルスシグナルとして収集する。

**動機:**
- 既に `top -l 1 -n 0` を呼んでいるので追加コストなし
- CPU 使用率だけでは捉えられない「重さ」を表現できる
- 「646プロセスも動いてて大変」のような Rebecca らしい表現が可能

**関連:** Phase 2+ のヘルスダッシュボード拡張

### [2026-02-13] ヘルス状態に応じた背景色の微細シフト

**概要:**
Overall Score に応じてページの `--bg` / `--surface` をごく微かに暖色系にシフトさせる。health が悪い時、訪問者がダッシュボードを見る前に「なんとなく違う」と感じるサブリミナルデザイン。

**動機:**
- [PRESENCE](concept/PRESENCE.md) の「生命感」を視覚の閾値以下で表現

- 技術的にはCSS変数の動的変更で実現可能

**課題:**
- 変化量が大きすぎると鬱陶しい
- 色覚障害対応の検討
