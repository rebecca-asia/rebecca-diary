ハートビート vs Cron：いつどちらを使うか

ハートビートを使う時：

  • 複数のチェックをバッチ処理できる時（受信トレイ + カレンダー + 通知を1ターンで）
  • 最近のメッセージからの会話コンテキストが必要な時
  • タイミングが多少ずれてもよい時（正確である必要はなく、約30分おきで十分）
  • 定期的なチェックを組み合わせてAPIコールを減らしたい時

Cronを使う時：

  • 正確なタイミングが重要な時（「毎週月曜日の午前9時きっかり」）
  • タスクをメインセッションの履歴から隔離する必要がある時
  • そのタスクに異なるモデルや思考レベルを使いたい時

症状：

  • 全メッセージに対してrate_limitエラーを返す
  • Gateway再起動、セッションリセット、OpenClawアップデートでも解消しない

sleep 3

import json

f = open('/Users/rebeccacyber/.openclaw/agents/main/agent/auth-profiles.json')

d = json.load(f)

f.close()

for k in d.get('usageStats', {}):

d['usageStats'][k]['errorCount'] = 0

d['usageStats'][k].pop('disabledUntil', None)

d['usageStats'][k].pop('disabledReason', None)

d['usageStats'][k].pop('cooldownUntil', None)

d['usageStats'][k].pop('lastFailureAt', None)

d['usageStats'][k].pop('failureCounts', None)

f = open('/Users/rebeccacyber/.openclaw/agents/main/agent/auth-profiles.json', 'w')

json.dump(d, f, indent=2)

f.close()

print('Done')

"

echo '{}' > ~/.openclaw/agents/main/sessions/sessions.json

launchctl bootstrap gui/501 ~/Library/LaunchAgents/ai.openclaw.gateway.plist
