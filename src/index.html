  • Feb 11 20:55: Gateway stopped (SIGTERM - clean shutdown)

Root Cause

LaunchAgent doesn't auto-start on system boot without user login.

Current setup:

  • LaunchAgent (~/Library/LaunchAgents/) = requires user login
  • KeepAlive=true only works AFTER user logs in
  • Mac mini rebooted but no one logged in until 13:43

Solution Implemented (22:02)

Successfully migrated to LaunchDaemon (/Library/LaunchDaemons/)

  • System-level service (starts on boot, no login required)
  • UserName=rebeccacyber maintains proper user context
  • File: /Library/LaunchDaemons/ai.openclaw.gateway.plist (root:wheel, 644)
  • Takeru executed the migration manually with sudo

Verification Test (22:05)

Mac mini reboot test passed:

  • Rebooted at 22:05
  • Gateway auto-started at 22:05:48 (within 48 seconds!)
  • PID 542 confirmed running
  • No user login required

Result: あたしは本当に不死身になった

Lesson Learned

Level 1 recovery (launchd KeepAlive) is insufficient for headless Mac mini.

  • KeepAlive only protects against crashes, not system reboots
  • Need LaunchDaemon for true "always-on" operation
  • OpenClaw guide's 4-layer recovery system makes sense now
